# 量化流动性管理 AI Agent:我们的设计原则与思路

> X(Twitter)长文草稿。发布前替换:仓库链接、联系方式。
> 定位:对外介绍设计原则与思路,不含收益承诺、不含实盘业绩声明、不讲内部迭代历史。

---

我们正在 Sui 上构建一个开源的量化流动性 AI Agent——**为 [LeafSheep](https://app.leafsheep.xyz) 生态打造的第三方托管 agent**。

先交代背景。LeafSheep 把你的 Cetus DLMM 仓位放进一个 Position Manager 合约,让同一份流动性赚三份收益:交易手续费、调仓低买高卖的价差、远端闲置 bin 存进借贷协议的利息。它的关键设计是 **Agent Delegation**:用户一个签名,就能把 PM 委托给任何能签 Sui 交易的 AI agent 来执行策略——而合约保证,无论委托给谁,**提现永远只认你自己的签名**。

我们做的,就是这个委托插槽里的"任何 AI agent"中认真的那一个:用机器学习预测短期价格分布,决定流动性怎么布、布多宽、什么时候撤;闲置资金自动进借贷吃利息;全程跑在 LeafSheep 的合约权限边界内——**碰得到仓位,碰不到提现**。

这篇文章讲清楚它的设计原则:为什么 LP 是个预测问题、预测什么、怎么把预测变成安全的链上操作、以及为什么我们的模型验收标准里没有 PnL。

如果你做链上做市、量化,或者想要一个能接自己模型的开源框架,这篇值得读完。

---

## 一、LP 的本质是一个预测问题

DLMM(离散流动性做市)把流动性切进一个个离散的 bin,价格每穿过一个 bin,该 bin 的挂单按占比成交。LP 的全部博弈浓缩成一个问题:**未来半小时,价格会在哪些 bin 里活动?**

市面上大多数自动调仓工具对这个问题的回答是:不知道,所以盯着当前价格,走了就跟着搬。这是"事后追价"——价格向下穿透你的 bid,你接了一堆货,然后才把仓位中心挪下去。每一次搬仓都在为已经发生的行情买单。

我们选择正面回答这个问题,而且回答方式有讲究:**不预测一个价格点,预测一个分布。**

模型(LightGBM 分位数回归)输出未来 30 分钟价格的三个分位数 q10 / q50 / q90,三个数派生出全部决策输入:

- **中心**(q50)→ 仓位围绕哪里布
- **宽度**((q90−q10)/2.56)→ 布多宽
- **穿越概率**(p_above / p_below)→ 要不要提前避险

价格预测是概率性的,所以仓位也应该是概率性的:按预测分布的形状给每个 bin 分配权重,而不是赌一个区间。预测说"大概率向下震荡后回归",bid 侧就自动加厚——**事前布仓,代替事后追价**。

这里有一个支撑整个设计的认知:**对 LP 来说,σ(波动率)比 μ(方向)重要得多,也好预测得多。** 30 分钟的方向预测接近抛硬币,我们对它的要求只是"统计上显著好于随机";但宽度预测错了,要么无常损失爆炸、要么吃不到手续费——这才是 LP 的生死线。所以系统天然支持一条保守路线:即使方向信号弱,只用模型的宽度和穿越概率,中心保持当前价格,仍然显著优于固定参数的规则做市。

## 二、架构原则:先问延迟预算,再选技术栈

整个系统两条工具链:**TypeScript(框架、决策、链上执行、风控)+ Python(训练、推理、回测)**,中间用一个本地 sidecar 服务连接。几个原则:

**1. 训练和推理共享同一份特征代码。**

ML 生产系统最隐蔽的失效模式是 train/serve skew:训练侧和推理侧的特征实现有细微差异,模型在线上悄悄变笨,没有报错、没有告警。根治办法只有一个——特征工程只实现一遍。我们的训练管线和推理服务 import 的是同一个 Python 模块,这类 bug 在结构上不存在。

**2. 性能优化对着真实的瓶颈,而不是对着简历。**

决策频率是每个池子 15–30 分钟一次,延迟预算是分钟级。在这个工作负载下,推理是 1ms 还是 100ms 没有任何区别——所以我们不为推理路径引入额外的编译型语言和原生绑定,把工程复杂度留给真正需要的地方。什么时候重新评估?决策频率进入秒级的那天。架构上留了口子(见第 5 条),换实现不动框架。

**3. 离散状态要少,连续参数要多。**

市场状态机只有三个状态,因为只有三种**行为模式**真正不同:

- **NORMAL**:对称做市。做市半宽不是档位,是预测宽度的连续函数——模型说波动大,仓位自动铺宽;模型说不确定性高,中心自动贴回当前价格。
- **TREND**:趋势市。这里有个关键设计:**趋势中不使用预测中心**——趋势里的"重心"是移动靶,跟着它布仓等于追涨杀跌。改用单侧偏置,偏置强度是穿越概率差的连续函数;强趋势时收缩成小规模反向仓位(押回调),其余资金进借贷。
- **EXTREME**:极端行情。全撤,100% 资金进借贷吃利息,每分钟重新评估,带滞回和最短驻留防止在临界点反复横跳。

凡是能用连续函数表达的,绝不做成离散档位——每多一个档位就多一组要标定的参数,而参数是过拟合的温床。整个状态机的自由度被刻意压在十来个,少到可以在影子数据上诚实地做网格搜索。

**4. 经济学约束是一等公民。**

链上做市每次调仓都有真实成本:gas + 服务扣费。模型再准,调仓太频繁也是亏的。所以决策层有两层成本保护:单笔层面,每个操作的期望收益必须超过 gas 成本的固定倍数,否则跳过;状态层面,每个状态都有一张预算表——期望日成本必须低于期望日手续费收入的一半,而手续费收入用**链上真实成交事件**标定,不用中心化交易所的成交量外推(两者差着一到两个数量级,直接外推会系统性高估收益)。

**5. 模型是可插拔的,框架不绑定任何 alpha。**

TS 框架只认识一个接口:`PredictionProvider`——输入市场快照,输出预测分布。我们的 LightGBM sidecar 是它的一个实现,纯规则的统计估计是另一个实现。换你自己的模型、远端服务、甚至另一种语言的推理引擎,都只是换一个实现,框架一行不动。

## 三、托管产品的安全设计:风控是产品本体,模型是可换的零件

这是个托管 agent,管的是用户的钱。两条设计红线:

**第一条:权限边界在合约层,不在代码层。**

Agent 通过 LeafSheep 的 CDPM 合约(PositionManager)操作,链上 allowlist 校验每一次调用。合约硬性保证:被委托的 agent 能加减流动性、收手续费、存取借贷协议,但**提不走一分钱**——提现、关仓、修改授权,只接受一个签名者:你。一币都不行,转去任何钱包都不行。你不需要信任我们的代码,你只需要读那份 Move 合约。

委托和撤销同样干净:一个签名把 PM 交给我们,委托后 LeafSheep 自己的策略 worker 会永久退出这个 PM——**同一份流动性上永远不会有两套策略打架**;你随时可以撤销授权,agent 立刻失去全部操作权限,我们的运行时监听到链上撤销事件后会自动把这个 PM 从监控里删除。

**第二条:风控的开发优先级高于模型。**

在我们的实施排期里,熔断系统是第一周的交付物,模型是第四周才接入的。逻辑很简单:对托管产品,**熔断、审计日志、降级路径才是产品本体,模型只是随时可以拔掉的 alpha**。

风控分三层:

- **L1 软熔断**:轻度异常信号(如跨市场价差异常)→ 提高借贷比例、收窄做市范围,降低暴露但不撤
- **L2 硬熔断**:闪崩、池子 TVL 骤降、价差严重失锚、模型预测穿越概率爆表 → 进 EXTREME,全撤 + 100% 借贷
- **L3 紧急停机**:疑似密钥泄漏、合约攻击 → 停止一切链上操作,只保留监控

再加一条贯穿全部的纪律:**没有静默降级**。模型超时、特征分布漂移(PSI 持续监控)、输入数据陈旧——每一次回退到规则策略都带着原因写进数据库,降级占比超过阈值直接告警。任何"预测失败但没留下记录"的路径,我们定义为 bug。

模型挂了会怎样?什么都不会发生。纯规则策略(统计波动率 + 概率分布布仓 + 趋势偏侧)作为兜底永远在线。**模型问题永远不会让 agent 停摆,更不会触及资金安全。**

## 四、验证哲学:验收标准里没有 PnL

这可能是全文最反直觉的一条:我们给模型设的上线标准里,**没有"回测收益跑赢多少"这一项**。

两个原因:

**回测的绝对收益不可信。** DLMM 回测的成交模拟依赖对链上交易流的假设,而任何代理数据源(比如中心化交易所成交量)和真实链上流都差着数量级。在一个有偏的模拟器上"跑赢基线 20%"不说明任何事情。所以回测在我们这里只做一件事:**同一个模拟器下的相对排序**——新策略和老策略谁更好,而不是新策略能赚多少。

**短窗口 PnL 是噪声。** 小资金跑几天的盈亏,方差完全淹没信号。用它做上线 gate,等于抛硬币。

那用什么验收?**可证伪的指标**:

- **预测质量**:pinball loss 必须显著优于统计基线;预测的 80% 区间,实际覆盖率必须落在 76–84%——一个说真话的模型,它说"80% 概率在这个范围",就真的应该有 80% 的时候在;方向准确率必须统计显著(binomial test),不显著就走"只用宽度"的保守路线。
- **单位经济学**:每个状态的成本预算表必须过关(见上文)。
- **影子模式**:模型先以"只记录、不执行"的方式陪跑两周,在线表现与离线训练一致(覆盖率偏差小于 5 个百分点)、降级占比达标、状态切换频率落在健康区间——全部通过才碰真钱,首期实盘上限刻意压到极小。

PnL 去哪了?降级为观察指标,实盘累计足够长的窗口后才参与下一版迭代的决策。

**不达标就不上线,退回去重训。这条规则本身也写在公开文档里。**

## 五、开源:我们给管线,alpha 你自己练

最后说开源定位,一句话:**仓库交付框架、风控和一条完整可复现的 ML 管线——但不交付训练好的模型。**

为什么?诚实的原因:做市 alpha 公开就失效。务实的原因:你也不应该信任一个陌生人训练的模型权重来管你的钱。

所以 fork 拿到的是:

- **完整的执行框架**:LeafSheep CDPM 权限边界内的原子调仓交易、闲置资金借贷路由(Scallop / Kai,APY 感知 + 单协议集中度上限)、用户充值与按次扣费的记账层
- **三层熔断风控 + 影子模式 + PnL 归因**,开箱即用
- **完整可复现的训练管线**:历史数据采集回填、特征工程、walk-forward 验证(防前视偏差)、推理 sidecar——固定随机种子,数据窗口和代码版本写进模型元数据,任何人都能复现训练过程
- **五个明确的扩展接缝**:策略、交易池、借贷协议、币种白名单、以及最关键的 `PredictionProvider`——接自己的模型只需要实现一个接口

LLM、新闻信号、跨链,都不在框架里,也不会进框架——要接,走接口装饰器或 sidecar 内部。**框架的价值是它保持小;策略和模型才是该生长的地方。**
